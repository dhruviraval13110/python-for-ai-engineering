# Conditions

## Overview

Conditional statements control the flow of a program by executing different blocks of code based on whether conditions evaluate to `True` or `False`.

---

## Topics Covered

- if
- if-else
- if-elif-else
- Nested if
- Conditional (ternary) expressions
- match-case
- Input validation

---

## Learning Objectives

After completing this module, you will be able to:

- Write decision-making logic.
- Combine conditions using logical operators.
- Choose between `if-elif` and `match-case`.
- Validate user input.
- Reduce nested logic using cleaner patterns.

---

## Best Practices

- Keep conditions simple and readable.
- Avoid deeply nested `if` statements.
- Use descriptive variable names.
- Prefer early returns in functions when appropriate.
- Use `match-case` for matching fixed values (Python 3.10+).

---

## Common Mistakes

- Using `=` instead of `==`
- Forgetting indentation
- Writing unnecessary nested conditions
- Overusing `elif` when a dictionary or `match-case` is more suitable
